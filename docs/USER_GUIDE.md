# 自动化获客系统功能与使用简介

## 系统定位

这套系统用于海外销售团队的自动化获客和客户生命周期管理。

核心目标：

- 获取新线索
- 找到客户工作邮箱
- 生成客户画像
- 发送外联邮件
- 跟踪打开、点击、退信、退订和回复
- 把客户从陌生线索推进到沟通、约会、试订单、代理协议和签约维护

系统第一版重点支持销售团队真实使用，不追求完全无人值守。高风险动作，例如正式发信、采用候选邮箱、推进生命周期，建议由销售或管理员确认后执行。

## 主要功能

### 1. 登录与权限

系统支持账号登录。

- 销售账号只能看到自己名下客户
- 管理员可以查看全部客户、团队日报、用户配额和发件账号
- 每个账号有每日获客配额和发信配额

### 2. 获取线索

系统提供多条获客路径。

#### Prospeo 自动获客

适合按公司网站、职位、行业、地区找客户。

填写：

- 目标公司网站
- 目标职位
- 行业
- 地区
- 数量

点击 `开始自动获客` 后，系统会把找到的联系人加入客户列表。

#### LinkedIn 公网搜索

适合通过 Google 公开索引搜索 LinkedIn 个人主页。

特点：

- 不登录 LinkedIn
- 不抓取 LinkedIn 后台页面
- 只搜索公开索引到的 `linkedin.com/in/*` 页面
- 自动解析姓名、职位、公司和地区
- 自动评分
- 自动补公司域名
- 自动生成邮箱候选
- 高分线索才触发少量付费验证

填写：

- 目标职位
- 行业关键词
- 地区
- 公司关键词
- 数量

点击 `开始公网搜索` 后，结果会进入搜索任务和客户列表。第一版不会自动发邮件。

#### CSV 导入

适合导入销售已有名单。

CSV 不强制要求 LinkedIn URL。有邮箱的客户会直接进入可富化/可发送流程；没有邮箱的客户后续可以点击 `邮箱` 富化。

#### 手动新增

适合临时录入单个客户，或测试完整流程。

可填写：

- LinkedIn URL
- 姓名
- 邮箱
- 职位
- 公司
- 公司域名

### 3. 邮箱富化

客户没有邮箱时，可以点击客户列表里的 `邮箱`。

系统会按配置的数据源顺序尝试：

- Prospeo
- Hunter
- NinjaPear
- GitHub commits
- 邮箱格式推断
- Gravatar
- 公司官网公开邮箱

规则：

- 只有 `personal_work + valid` 邮箱会作为正式发信邮箱
- `info@`、`sales@`、`support@`、`contact@` 等通用邮箱只作为候选，不自动发信
- LinkedIn 公网搜索生成的候选邮箱可以在客户详情里人工采用

### 4. 社媒与客户资料补全

点击客户列表里的 `社媒`，系统会尝试补全客户相关公开资料。

可用于：

- 找 LinkedIn、Twitter/X、GitHub、Facebook、官网等主页
- 辅助 AI 画像
- 辅助销售判断客户质量

### 5. AI 客户画像

点击客户列表里的 `画像`，系统会基于已有线索、邮箱、社媒和阶段记录生成客户画像。

画像包含：

- 客户匹配度
- 客户背景摘要
- 可能兴趣点
- 可能痛点
- 意向判断
- 下一步动作建议

画像会显示在客户列表和客户详情里。

### 6. 邮件发送

发送前客户需要满足：

- 有正式有效邮箱
- 不在黑名单
- 已加入队列或直接手动发送
- 未超过用户每日发信配额
- 未超过全局每日发信配额

发送方式：

- 客户列表点击 `入队`
- 客户列表点击 `发送`
- 客户详情里使用自定义邮件或 AI 生成邮件后发送

系统会记录：

- 发件账号
- 联系人
- 邮件 step
- 发送时间
- 打开、点击、退信、退订等事件

### 7. 邮件反馈与今日待办

首页会显示今日待办，优先处理：

- 已打开未回复
- 已回复
- 退信需处理

客户列表会展示：

- 已发送次数
- 打开次数
- 点击次数
- 是否回复
- 是否退信
- 是否退订

销售可以据此决定是否二次触达、等待、放弃或推进客户生命周期。

### 8. 客户生命周期管理

系统内置客户生命周期漏斗：

- 线索
- 回复
- 初步沟通
- 约会/会议
- 商业计划
- 试订单
- 代理协议
- 门店创建
- 签约维护
- 等待池
- 放弃

在客户列表中可以点击：

- `推进`
- `等待`
- `放弃`
- `详情`

在客户详情里可以记录：

- 客户回复
- 背景调研
- 会议纪要
- 商业计划
- 试订单信息
- 协议风险
- 门店资料

每条阶段记录都可以让 AI 分析，生成下一步建议、风险点和准备材料。

### 9. 管理员功能

管理员可以：

- 新增销售账号
- 重置密码
- 启用/停用账号
- 调整每人每日获客配额
- 调整每人每日发信配额
- 查看团队运营日报
- 查看邮箱 Provider 命中情况
- 查看发件账号池
- 查看失败原因统计

## 每日推荐使用流程

### 销售日常流程

1. 登录系统
2. 在 `获客操作` 中通过 Prospeo、LinkedIn 公网搜索、CSV 或手动新增获取客户
3. 对没有邮箱的客户点击 `邮箱`
4. 对重要客户点击 `社媒` 和 `画像`
5. 有有效邮箱后点击 `入队`
6. 检查邮件内容，确认后点击 `发送`
7. 每天查看 `今日待办`
8. 对已打开未回复客户做二次触达
9. 对回复客户进入客户详情，记录沟通内容并推进生命周期
10. 对退信、退订或明显不匹配客户标记放弃

### 管理员日常流程

1. 查看团队运营日报
2. 查看每个销售的获客数、有效邮箱数、发信数和剩余额度
3. 查看 Provider 命中率和失败原因
4. 检查退信率和投诉情况
5. 控制发件账号 warmup 节奏
6. 根据数据调整销售配额和数据源策略

## 注意事项

- 不要直接对未验证邮箱大批量发信
- 不要把公司官网通用邮箱当作个人销售线索邮箱
- 新发件域名和新账号需要 warmup，不要第一天直接满量发送
- 客户退订后不要继续发送
- 退信率过高时要暂停发信并检查邮箱质量
- LinkedIn 公网搜索只使用公开索引页面，不要登录抓取 LinkedIn 后台
- API key 和 `.env` 文件不要发到群里或提交到 Git

## 常见问题

### 为什么 LinkedIn 搜索能找到人但没有邮箱？

LinkedIn 页面通常不公开邮箱。系统会先解析姓名、职位、公司和域名，再生成邮箱候选，并用 Hunter/Prospeo 等服务验证。没有验证通过时，不会自动写入正式邮箱。

### 为什么有些邮箱显示为候选？

候选邮箱还没有达到正式发信标准。销售或管理员可以在客户详情里查看来源、置信度和状态。只有个人工作邮箱才建议采用。

### 为什么不能直接每天每人发 100 封？

新域名和新发件账号需要预热。建议先从每账号每天 10-20 封开始，再逐步提升。否则容易退信、进垃圾箱或影响域名信誉。

### 为什么生产自检里 public_url 本地不通过？

本地地址是 `http://127.0.0.1:8765`，不是公网 HTTPS。部署到云服务器并配置 `https://sales.frelys.xyz` 后才会通过。

### 什么情况下可以正式开放给销售？

建议满足：

- `doctor --strict` 通过
- 域名 HTTPS 可访问
- Resend webhook 已配置
- 发件域名 DKIM/SPF/DMARC 正常
- 管理员强密码已设置
- 备份正常生成
- 小流量真实发送测试成功
